Contribution Rules and Guide

Rules:

  • Follow the project's code of conduct and guidelines.
  • Create an issue for the proposed contribution before making changes.
  • Fork the repository and work on a new branch for your contribution.
  • Keep code clean, organized, and well-documented.
  • Test your changes thoroughly before submitting a pull request.
  • Adhere to the project's style guide and coding conventions.
  • Be respectful and collaborative with other contributors.

Contribution Guide:

  1. Fork the repository on GitHub.
  2. Clone the forked repository to your local machine: git clone https://github.com/your-username/repository.git
  3. Create a new branch for your contribution: git checkout -b my-contribution
  4. Make changes, add new features, or fix bugs.
  5. Test your changes to ensure they work as intended.
  6. Commit your changes: git commit -m "Description of your changes"
  7. Push your changes to your forked repository: git push origin my-contribution
  8. Create a pull request on GitHub from your forked repository.
  9. Describe your contribution and submit the pull request.
  10. Collaborate and discuss with the project maintainers for further improvements if needed.
